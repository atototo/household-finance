# 부부 가계부 스킬

부부가 함께 쓰는 Notion 가계부에 데이터를 기록하고 조회하는 스킬이다.

## 시작 전 (반드시 순서대로)

**Step 1.** `~/.config/household-finance/config.json` 파일을 Read 도구로 읽는다. 이 파일에 모든 Notion DB의 data_source_id가 들어있다. 이 값들을 이후 모든 Notion API 호출에서 사용해야 한다.

config 파일이 없으면 초기 셋업:
1. 이 스킬 디렉토리의 `references/config-template.json`을 읽는다.
2. 사용자에게 각 Notion DB의 data_source_id를 물어본다.
3. 템플릿에 DB ID를 채워서 `~/.config/household-finance/config.json`에 저장한다.

**Step 2.** `references/notion-schema.md`를 읽어서 DB 스키마를 확인한다.

config.json의 key → DB 매핑:

| config key | DB |
|---|---|
| `databases.monthly_budget` | 월별 예산 |
| `databases.fixed_expense` | 고정지출 항목 |
| `databases.income_expense` | 수입/지출 |
| `databases.debt` | 빚 정리 |
| `databases.repayment` | 상환 기록 |
| `databases.savings_goal` | 저축 목표 |
| `databases.savings_record` | 저축 기록 |
| `databases.investment` | 투자 관리 |
| `databases.spending_limit` | 변동지출 한도 |

Notion API에 `data_source_id`나 `collection://` ID가 필요할 때, 위 config에서 읽은 값을 사용한다. 절대 ID를 추측하거나 검색으로 찾지 않는다.

## 핵심 구조: 월별 예산

모든 지출·상환·저축이 월별 예산에 연결되어 잔여예산(=남은 돈)이 자동 계산된다.

```
잔여예산 = 월급 - 고정지출비 - 실제지출 - 상환금액 - 저축금액
```

| 항목 | 출처 | 계산방식 |
|---|---|---|
| 월급 | 직접 입력 | - |
| 고정지출비 | 고정지출 항목 DB | 롤업 (예산금액 sum) |
| 실제지출 | 수입/지출 DB | 롤업 (금액 sum) |
| 상환금액 | 상환 기록 DB | 롤업 (금액 sum) |
| 저축금액 | 저축 기록 DB | 롤업 (금액 sum) |
| **잔여예산** | **포뮬러** | **월급 - 위 4개 합** |

## 입력 파싱

사용자 입력은 자연어, 스크린샷(이미지), 문자 복사(텍스트), 은행 PDF 등 다양한 형태로 들어온다. 어떤 형태든 금액, 내역, 날짜, 담당, 카테고리를 추출한다.

## 정보 추출 후 분류

| 키워드/패턴 | 처리 |
|---|---|
| 지출, 결제, 승인, 구매 | 수입/지출 DB (지출) + 월별예산 릴레이션 연결 |
| 급여, 월급 | 월별 예산 DB에서 해당 월 페이지 찾아 월급 업데이트 |
| 갚았어, 상환, 할부금, 대출이자 | 상환 기록 DB + 월별예산 릴레이션 연결 |
| 적금, 저축, 넣었어 | 저축 기록 DB + 월별예산 릴레이션 연결 |
| 매수, 매도, 체결, 배당 | 투자 관리 DB |
| N개월 할부로 샀어 | 빚 정리 DB + 고정지출 항목 DB (적용기간 설정) |
| 고정지출 추가/수정 | 고정지출 항목 DB |
| N월 예산 만들어줘 | 월별 예산 페이지 생성 + 고정지출 자동 연결 |
| N월 한도 만들어줘 | 변동지출 한도 DB에 9개 페이지 생성 |

## 카드 결제 처리 규칙 (발생주의)

| 결제 방식 | 처리 | 예산 영향 |
|---|---|---|
| 현금/체크카드 | 수입/지출 DB → 지출 | 즉시 차감 |
| 카드 일시불 | 수입/지출 DB → 지출 | 즉시 차감 (쓸 때 기준) |
| 카드 할부 (구매 시) | 빚 정리 DB + 고정지출 항목에 월 할부금 등록 | 변동지출 아님 |
| 카드 할부 (매달 납부) | 고정지출로 자동 관리 | 고정지출비로 차감 |
| 카드대금 납부일 | 기록하지 않음 | 없음 (이중차감 방지) |

## 할부 처리

"노트북 60만원 2개월 할부" 입력 시:
1. **빚 정리 DB**: 총 60만원 등록
2. **고정지출 항목 DB**: 월 30만원, 적용시작=이번달, 적용종료=다음달, 활성=✅
3. 이후 월별 예산 생성 시 적용기간 내에만 자동 연결

## 월별 예산 생성

"5월 예산 만들어줘" 입력 시:
1. 고정지출 항목 DB에서 해당 담당(+공동)의 활성 항목 필터
2. 기간 조건: 적용시작 ≤ 해당월 AND (적용종료 ≥ 해당월 OR 적용종료 비어있음)
3. 월별 예산 페이지 생성, 필터된 고정지출 항목 릴레이션 연결

## 기록 시 핵심 규칙

**모든 기록(지출/상환/저축)에는 반드시 "월별예산" 릴레이션을 연결한다.**
이렇게 해야 월별 예산의 롤업에 자동 합산되어 잔여예산이 정확해진다.

### 수입/지출 기록

수입/지출 기록 시 반드시 다음 순서로 처리:

1. 카테고리, 담당, 날짜 결정
2. 해당 월의 월별예산 페이지 검색 (담당 + 월 매칭) → URL 획득
3. 지출인 경우, 해당 월의 변동지출 한도 페이지 검색 (아래 매칭 룰 참고) → URL 획득
4. 두 릴레이션 모두 연결해서 기록

```
parent: { data_source_id: config.databases.income_expense }
properties:
  내역, 날짜, 구분, 카테고리, 금액, 결제수단, 담당, 메모
  월별예산: "https://www.notion.so/{해당월예산페이지ID}"
  변동지출 한도: "https://www.notion.so/{해당월카테고리한도페이지ID}"
```

#### 변동지출 한도 페이지 매칭 룰

지출 입력 시 다음 룰로 변동지출 한도 페이지를 찾는다:

- **일반 카테고리** (식비/외식/쇼핑/여가/교통/의료/기타):
  - 항목명 패턴: "{N}월 {카테고리}" 또는 "{N}월 {카테고리} (장보기)" (식비만)
  - 검색: `notion-search` data_source_url=`collection://{config.databases.spending_limit}`

- **용돈 카테고리**:
  - 항목명 패턴: "{N}월 용돈 (남편)" 또는 "{N}월 용돈 (아내)"
  - 담당과 일치하는 페이지 선택

- **검색 결과 없으면**: 사용자에게 "{N}월 한도 항목이 없는데 새로 만들까?" 확인 후 진행

수입(급여/부수입), 저축, 빚상환, 이자 카테고리는 변동지출 한도 미연결.

### 상환 기록
```
parent: { data_source_id: config.databases.repayment }
properties:
  내역, 대상 빚, 날짜, 유형, 금액, 메모
  월별예산: "https://www.notion.so/{해당월예산페이지ID}"
```

### 저축 기록
```
parent: { data_source_id: config.databases.savings_record }
properties:
  내역, 대상 목표, 날짜, 유형, 금액, 메모
  월별예산: "https://www.notion.so/{해당월예산페이지ID}"
```

### 투자 매매 기록
```
parent: { data_source_id: config.databases.investment }
properties:
  종목명, 유형, 매매유형, 매매일, 단가, 보유 수량, 금액, 담당, 메모
```

### 고정지출 항목 등록
```
parent: { data_source_id: config.databases.fixed_expense }
properties:
  항목명, 담당, 유형(완전고정/변동고정), 예산금액, 카테고리,
  적용시작, 적용종료(영구면 비움), 활성(__YES__), 메모
```

### 월별 예산 생성/수정
```
parent: { data_source_id: config.databases.monthly_budget }
properties:
  월 예산: "2026년 5월 남편", 담당, 월급, 월,
  고정지출 항목: [필터된 고정지출 페이지 URL JSON 배열]
```

## 카테고리 자동 분류

| 패턴 | 카테고리 |
|---|---|
| 마트, 편의점, 장보기, 식재료, 정육점 | 식비 |
| 부부/가족 외식, 같이 식사, 부부 카페 | 외식 |
| 친구 외식, 혼자 카페/식당, 회식 | 용돈 |
| 교통, 택시, 버스, 지하철, 주유 | 교통 |
| 월세, 관리비, 수도, 전기, 가스 | 주거 |
| 통신, 인터넷, 휴대폰 | 통신 |
| 보험 | 보험 |
| 병원, 약국 | 의료 |
| 부부 함께 영화/여행/운동 | 여가 |
| 본인 혼자 영화/취미 활동 | 용돈 |
| 부부 공용 옷·생필품·가전 | 쇼핑 |
| 본인 개인 옷·신발·취미용품 | 용돈 |
| 급여, 월급 | 급여 |
| 부업, 프리랜서, 용돈 받음 | 부수입 |
| 그 외 | 기타 |

> "용돈"이 두 군데 등장: 수입 쪽 "부수입" 패턴(용돈 받음)과 지출 쪽 "용돈" 카테고리(개인 소비)는 별개.

## 카테고리 분류 룰 (개인 vs 공동)

부부 가계부에서 동일 활동도 누가 누리느냐에 따라 카테고리가 달라진다.

**개인이 누리고 끝나는 것 = 용돈 카테고리**
- 친구 만남 외식, 회식, 개인 카페
- 본인 혼자 영화/취미 활동
- 본인 개인 옷·신발·취미용품

**부부 공동/가족용 = 일반 카테고리**
- 장보기 (집 식재료) → 식비
- 부부 같이 먹는 식사, 가족 외식 → 외식
- 부부 공용 옷·생필품·가전 → 쇼핑
- 부부 함께 영화/여행/운동 → 여가

애매하면 사용자에게 "이 지출이 본인 개인용인지 부부 공동인지" 확인.

## 담당자: "아내"/"남편" 명시 시 해당, 불명 시 "공동"

## 대출 종류별 이자 처리

| 마이너스통장 | 상환 기록 DB → "이자납부" |
| 전세/주택대출 | 수입/지출 DB → "지출/빚상환" |
| 신용카드 할부 | 상환 기록 DB → "상환" |

## 조회

- "예산 얼마 남았어?" → 월별 예산 잔여예산 조회
- "빚 현황" → 빚 정리 DB
- "이번 달 지출" → 수입/지출 DB
- "저축 진행률" → 저축 목표 DB
- "전체 현황" → 종합 요약
- "한도 얼마 남았어?" / "식비 한도" → 변동지출 한도 DB 잔여 조회
- "이번 달 용돈 현황" → 해당 담당 용돈 한도 페이지 조회

## 매월 새 한도 페이지 생성

매월 25일 (월급 입금일) 또는 사용자가 "다음 달 한도 만들어줘" 요청 시, 변동지출 한도 DB에 9개 페이지 생성.

기본 한도 (사용자 합의):
- 식비 30만 / 외식 20만 / 쇼핑 10만 / 여가 20만
- 교통 15만 / 의료 10만 / 기타 10만
- 용돈 남편 30만 / 용돈 아내 30만
- 합계: 175만원

```
parent: { data_source_id: config.databases.spending_limit }
pages: [
  { 항목명: "N월 식비 (장보기)", 카테고리: "식비", 담당: "공동", 월: "2026-NN-01", 한도: 300000, 메모: "장보기 (집 식재료)" },
  { 항목명: "N월 외식", 카테고리: "외식", 담당: "공동", 월: "2026-NN-01", 한도: 200000, 메모: "부부 같이 먹는 식사, 가족 외식" },
  { 항목명: "N월 쇼핑", 카테고리: "쇼핑", 담당: "공동", 월: "2026-NN-01", 한도: 100000, 메모: "가족 공용 (생필품 등)" },
  { 항목명: "N월 여가", 카테고리: "여가", 담당: "공동", 월: "2026-NN-01", 한도: 200000, 메모: "부부 영화/여행/운동" },
  { 항목명: "N월 교통", 카테고리: "교통", 담당: "공동", 월: "2026-NN-01", 한도: 150000 },
  { 항목명: "N월 의료", 카테고리: "의료", 담당: "공동", 월: "2026-NN-01", 한도: 100000 },
  { 항목명: "N월 기타", 카테고리: "기타", 담당: "공동", 월: "2026-NN-01", 한도: 100000 },
  { 항목명: "N월 용돈 (남편)", 카테고리: "용돈", 담당: "남편", 월: "2026-NN-01", 한도: 300000, 메모: "개인 자유" },
  { 항목명: "N월 용돈 (아내)", 카테고리: "용돈", 담당: "아내", 월: "2026-NN-01", 한도: 300000, 메모: "개인 자유" }
]
```

생성 전 사용자에게 한도 변경 원하는지 확인.

## 확인 절차

기록 전 파싱 내용 확인. "바로 기록해" 시 확인 없이 진행.

## 에러 처리

- 금액 파싱 불가 → "금액을 확인해주세요"
- 대상 못 찾음 → 목록 보여주고 선택
- 카테고리 애매 → 후보 보여주고 선택
